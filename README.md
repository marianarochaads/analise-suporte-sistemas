# analise-suporte-sistemas
Documentação e diagnóstico de incidentes em sistemas legados. Foco em resolução de problemas, análise de causa raiz e otimização de fluxo.
# Análise e Suporte a Sistemas Legados

## 1. O Problema
Sistemas críticos operavam com alta incidência de chamados recorrentes e falta de documentação sobre falhas conhecidas, gerando sobrecarga na equipe de TI e tempo de inatividade para o usuário final.

## 2. A Solução
Implementação de um método estruturado para registro e resolução de incidentes (Análise de Causa Raiz - RCA). Criação de uma base de conhecimento para problemas recorrentes e otimização de scripts de correção para o suporte técnico.

## 3. Tecnologias & Ferramentas
- Diagnóstico: Análise de Logs e SQL (Query Analysis)
- Documentação: Base de Conhecimento (KEDB)
- Processos: ITIL (Fundamentos)
- Ferramentas: Sistemas de Ticketing (ex: Jira / GLPI)

## 4. Destaques Técnicos
* **Diagnóstico de Causa Raiz:** Aplicação de lógica estruturada para identificar a origem dos erros, reduzindo a reincidência de chamados em 30%.
* **Documentação Técnica:** Criação de manuais de correção rápida (Knowledge Base) para padronizar o suporte.
* **Resiliência:** Melhoria nos processos de verificação de logs para identificar falhas antes que impactassem o ambiente de produção.

## 5. Resultados
- Redução significativa no tempo médio de atendimento (TMA) dos chamados.
- Maior estabilidade do sistema através da resolução definitiva de problemas crônicos.
- Padronização do atendimento técnico, aumentando a satisfação dos usuários.
